MAC OS 终端使用技巧总结

OS X系统为什么让人爱不释手？一是因为它的图形界面很美，二是它的命令行工具很强大，有完整的工具链供程序员使用。
在码代码的过程中，免不了大量得使用终端，如果不掌握下面说的这些技巧，效率就很难提高。

技巧1：control + u 清空命令行

如果你在一条终端命令中发现有输入错误的话，那么用 control+u 可以直接删除这一整条命令，然后你就可以重新输入。注意不是command，而是control。

技巧2：mkdir -p 创建多级目录

想创建多级目录，你还在不停的敲mkdir吗？一条mkdir就足以了，只需要加上-p参数。

技巧3：!! 加入sudo执行上一条命令

有时候，你写了一条长长的命令，按回车之后发现没有执行权限，需要在命令前添加 sudo，这时你可能要崩溃。不过其实可以用 sudo !! 这条命令来解决，它执行的效果和 sudo {上一条命令} 是一样的。

技巧4：history 列出执行过的命令 !index 重新执行编号为index的命令

用 history 命令可以显示你最近执行过的命令历史记录，你还可以指定显示条数，比如 history 20 即可显示最近的20条命令历史记录。此外，你还可以筛选包含特定字符的命令，比如用 history | grep mk，就可以只显示历史记录中包含 mk 的命令。
用!index可以重新执行history列出来的命令中编号为index的命令。

技巧5：&& 合并两条命令

&& 可以将两条命令合并成一条命令，让两条命令一起执行，不过是串行执行。

技巧6：reset 清空终端

reset 的作用很简单——将目前「终端」屏幕上的内容清空，就好像刚刚打开终端一样。

技巧7：bg fg jobs control+z nohup 轻松控制程序前/后台运行

要后台运行一个命令 command： command &
运行一个命令，终端关闭也继续运行：nohup command
后台运行一个命令，终端关闭也继续运行：nohup command &

一个任务已经在前台运行，control + z 挂起这个任务到后台，执行暂时中断
用bg %jobnumber 让任务在后台继续运行
用fg %jobnumber 让任务在前台继续运行
用jobs 查看当前后台运行的全部任务
jobs -l 可以连同显示PID

exit: 用exit退出终端时，如果当前有后台运行程序，会提示，比直接点叉叉关掉终端安全！！

作业在后台运行一样会将结果输出到屏幕上，干扰你的工作。如果放在后台运行的作业会产生大量的输出，最好使用下面的方法把它的输出重定向到某个文件中:
command >out.file 2>&1 &
其中，2>&1的意思是所有的标准输出和错误输出都将被重定向到一个叫做out.file 的文件中。

技巧8：chmod -w file 防止文件被无意中改写

对于那些永远都不应该被改动的文件，用这条命令防止被无意中改了找不回来。因为命令行太强大了，一个错误的命令就会导致意想不到的事情发生。
对于文件夹中所有的文件，递归操作的命令是：
chmod -R -w folder

技巧9：kill 杀死一个进程

用top -o cpu可以列出当前占用cpu最多的程序，得到PID
kill pid 杀死进程号为pid的进程
Kill -9 pid 强制杀死，因为强制杀死不给程序清空资源的机会，除非上一条命令不能用，尽量不适用

技巧10：alias 给常用命令设置别名

这个技巧真是太有用了，极大的提升了效率。
我常用的一些别名：
alias ls=”/bin/ls -aFG”
alias desktop=”cd ~/Desktop”
alias work=”cd ~/work”
alias home=”cd ~/“
alias /=”cd /“
alias bc=”open /Applications/Beyond\ Compare.app”
…
